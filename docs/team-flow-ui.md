## `/team` 命令的交互细节

### 方式 1: `/team` + 需求（一步到位）

```
用户输入:  /team 实现登录页面的表单验证
         ─────  ─────────────────────
         命令    需求描述

LLM 收到后:
1. 创建团队 (用默认团队模板)
2. 拆解任务
3. spawn teammates
4. 分配任务
5. 开始并行执行
```

### 方式 2: `/` 下拉选择团队 key（指定团队模板）

```
用户输入:  /
         ↓ 弹出下拉菜单
         ┌────────────────────┐
         │ 🔍 搜索...          │
         ├────────────────────┤
         │ 👤 Sisyphus  agent │  ← 切换到单个 agent
         │ 👤 Explore    agent │
         │ 👤 Oracle     agent │
         ├────────────────────┤
         │ 🏠 /default   team │  ← 选择团队模板
         │ 🏠 /ios-dev   team │  ← 选择 iOS 团队
         ├────────────────────┤
         │ ⚡ /build-ios skill│
         │ ⚡ /build-android  │
         └────────────────────┘

用户选择 /ios-dev 后:
→ Composer 显示 "使用 iOS 开发团队 模式"
→ 用户继续输入需求
→ 发送后 LLM 用 ios-dev 团队模板创建团队
```

### `/team` 不带参数

```
用户输入:  /team
         ↓
→ 使用默认团队模板 (key=default)
→ LLM 会问: "你想让团队做什么？"
→ 用户继续输入需求
```
